Ruby on Rails development environment with Docker
VA
Jan 23, 2016
2 minutes read

Learn Ruby and Rails

This was how I practiced stuff I learned about Ruby and Rails, I decided to create a web application for photo gallery. For starter, I wanted to keep my dev env more clean, reason why I opt to use Docker, I had it already installed, as well as Ruby and Rails.

I created the new project with rails new gallery --skip-bundle, (going to install bundle in the container) it added all the files and folder I need for the Rails project, then I added everything in to a Git repository, just to keep track of it.

Added new file called Dockerfile with the source I found on Marko Locher article:

    FROM ruby:2.3

    # Install apt based dependencies required to run Rails as
    # well as RubyGems. As the Ruby image itself is based on a
    # Debian image, we use apt-get to install those.
    RUN apt-get update && apt-get install -y \
      build-essential \
      patch \
      ruby-dev \
      zlib1g-dev \
      liblzma-dev \
      nodejs

    # Configure the main working directory. This is the base
    # directory used in any further RUN, COPY, and ENTRYPOINT
    # commands.
    RUN mkdir -p /app
    WORKDIR /app

    # Copy the Gemfile as well as the Gemfile.lock and install
    # the RubyGems. This is a separate step so the dependencies
    # will be cached unless changes to one of those two files
    # are made.
    COPY Gemfile ./
    RUN gem install bundler && bundle install --jobs 20 --retry 5

    # Copy the main application.
    COPY . ./

    # Expose port 3000 to the Docker host, so we can access it
    # from the outside.
    EXPOSE 3000

    # Configure an entry point, so we don't need to specify
    # "bundle exec" for each of our commands.
    ENTRYPOINT ["bundle", "exec"]

    # The main command to run when the container starts. Also
    # tell the Rails dev server to bind to all interfaces by
    # default.
    CMD ["bundle", "exec", "rails", "server", "-b", "0.0.0.0"]

Then built the app container docker build -t gallery . and then started it docker run -d --name gallery -v "$PWD":/app -p 8080:3000 gallery

Now every time I want to run any rails command, I’m going to use the docker, it will look like this docker run --rm -v "$PWD":/app gallery rails g controller Albums index to make things are easier for me in the console, I created an alias in my ~/.basj_profile with the following command alias dockerrun='docker run --rm -v "$PWD":/app' so now I can do this dockerrun gallery rails g controller Albums index
